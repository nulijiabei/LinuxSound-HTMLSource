The audio tag

HTML 5 is a "living standard". Uggh! That means it isn't a standard at all, but just a label for a specification that is in a state of flux. Nevertheless, most vendors involved in this have agreed that a new tag - the <audio> tag will belong to HTML 5 once it settles down.

For example, the following HTML will try the Ogg file first and if the client cannot play it, it will try the MP3 file and if it cannot play that then it will display the failure message: 

	
      <audio controls="controls"<
	<source src="audio_01.ogg" type="audio/ogg"<
	  <source src="audio_01.mp3" type="audio/mpeg"<
	    Your browser does not support the audio element.
      </audio<
